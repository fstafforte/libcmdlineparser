Command Line Parser Documentation

A Library that manages process command line options.
Options can be expressed in a short and long form or a long form only.
i.g. "myprocess --config-file=myconfigfile.cfg"

Four types of options are recognized:
1) Boolean options: i.g. "-v" or "--verbose" or "--verbose=true"
2) Integer options: i.g. "-i1000" or "-i 1000" or "--interval 1000" "--interval=1000"
3) Floating point options: i.g. "-t123.45" or "-t 123.45" or "--tollerance 123.45" or "--tollerance=123.45"
4) String options: i.g. "-cmyconfigfile.cfg" or "-c myconfigfile.cfg" or "--config-file myconfigfile.cfg" or "--config-file=myconfigfile.cfg"

In a command line the same option can be written more than one time
Use CommandLineParser::getOptionValues methods to retrive all the occurrences

Long form option can be reconized even if the user write it partialy (i.g. "--toll=123.45").
This can create ambiguity so the command line parser class check for that and throws a relevant exception.

While boolean options do not require a mandatory argument, the other options need one.
Options could be mandatory or not.
More over all option types could be mandatory or not.
